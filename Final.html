WebStack Fundamentals

Part-1: Introduction

What is a Network & Internet?

A network consists of two or more computers that are linked in order to share resources (such as printers and CDs), exchange files, or allow electronic communications.

The Internet is a global network of interconnected computers and devices that enables communication and data exchange across the globe. It serves as the backbone for transmitting information, supporting various online services, and facilitating real-time interactions.

Note:

The computers on a network may be linked through cables, telephone lines, radio waves, satellites, or infrared light beams.

Two very common types of networks include:

  • Local Area Network (LAN)
  • Wide Area Network (WAN)

Part-2: Network Architecture

Peer-to-Peer (P2P): Involves distributed systems communicating and sharing resources without a central authority.

Service-Oriented Architecture (SOA): Focuses on reusable services that can be combined to form applications.

Microservices Architecture: Breaks down applications into independently deployable services.

Client-Server Architecture: A distributed computing model where tasks are divided between service providers (servers) and service requesters (clients).

Client-Server Architecture Functionality

Clients initiate communication sessions with servers, send requests, and wait for responses. Servers wait for client requests, process these requests, and send back appropriate responses.

Advantages of Client-Server Architecture

  • Scalability: Servers can be upgraded or replicated to handle more clients.
  • Maintainability: Centralized management of resources and services makes maintenance easier.
  • Security: Centralized control allows for better implementation of security measures.
  • Resource Sharing: Efficient use of resources as multiple clients can share the same server resources.

Part-3: Communication Methods & Protocols

Client-server communication is typically facilitated by various network protocols such as HTTP (for web services), FTP (for file transfer), SMTP (for email), and SQL (for database access).

HTTP

HTTP (Hypertext Transfer Protocol) is the foundational protocol used on the World Wide Web for transmitting data between a client (typically a web browser) and a server.

Requests and Responses:

HTTP Request: Sent by the client to request data or perform an action on the server. It includes methods (GET, POST, PUT, DELETE, etc.), headers, and sometimes a body.

HTTP Response: Sent by the server in reply to the client's request. It includes status codes, headers, and often a body containing the requested data.

HTTP Methods

  • GET: Retrieves data from the server without causing any side-effects.
  • POST: Submits data to be processed to a specified resource.
  • PUT: Updates a resource on the server with new data or creates a new one.
  • DELETE: Removes the specified resource from the server.
  • HEAD: Retrieves only the headers, not the body of the response.
  • OPTIONS: Returns the HTTP methods that the server supports for a specific resource.
  • PATCH: Applies partial modifications to a resource.

HTTP Status Codes

HTTP status codes are issued by the server in response to a client's request. They are divided into five categories:

  • 1xx (Informational): 100 Continue, 101 Switching Protocols
  • 2xx (Success): 200 OK, 201 Created, 204 No Content
  • 3xx (Redirection): 301 Moved Permanently, 302 Found, 304 Not Modified
  • 4xx (Client Error): 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found
  • 5xx (Server Error): 500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable

Part-4: SSL (Secure Sockets Layer)

SSL (Secure Sockets Layer) is a standard security technology for establishing an encrypted link between a web server and a web browser. It ensures that all data passed between the web server and browsers remain private and integral. SSL has been succeeded by Transport Layer Security (TLS), but the term SSL is still commonly used.

How SSL Works

  1. Client Hello: The client sends a message to the server with its SSL version, cipher settings, session-specific data, and other information necessary for establishing a connection.
  2. Server Hello: The server responds with its SSL version, cipher settings, session-specific data, and its SSL certificate.
  3. Certificate Validation: The client verifies the serverâ€™s SSL certificate with a trusted Certificate Authority (CA).
  4. Key Exchange: The client generates a session key, encrypts it with the server's public key, and sends it to the server.
  5. Secure Connection: Once the handshake is complete, a secure, encrypted connection is established.

Key Features of SSL

  • Encryption: SSL uses encryption to protect data in transit between the client and server.
  • Authentication: SSL certificates verify the identity of the server.
  • Data Integrity: SSL ensures that data sent between the client and server is not tampered with during transit.

SSL Certificates

Types of SSL Certificates:

  • Domain Validated (DV): Basic level of validation, ensuring the domain is registered.
  • Organization Validated (OV): More rigorous validation process, including verifying the organization's identity.
